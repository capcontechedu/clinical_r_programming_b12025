4. Project-Oriented Workflow

When you work on R scripts, itâs tempting to save everything in one big file. But as projects grow, that becomes messy.

9. Best Practices

  • Always use RStudio Projects (keeps work isolated).

  • Always initialize renv for any project thatâs more than a quick script.

  • Commit renv.lock to Git (so others can reproduce).

  • Donât commit the renv/library folder (too heavy).

10. Recap in Layman Terms

  • Packages = toolkit that extend R.
  • Project workflow = keep everything for one project in its own folder.
  • Problem = package versions change over time, breaking old projects.
  • Solution = renv locks package versions, making projects reproducible.
