Variable Names

A variable can have a short name (like x and y) or a more descriptive name (age, carname, total_volume). Rules for R variables are:

  • A variable name must start with a letter and can be a combination of letters, digits, period(.) and underscore(_). If it starts with period(.), it cannot be followed by a digit.
last.name <- "Programming"
print(paste("last.name holds value =",last.name))
[1] "last.name holds value = Programming"
first_name <- "R"
print(first_name)
[1] "R"
# "." but can not follow by Number or "_"
.first <- "ICON"
print(.first)
[1] "ICON"

  • A variable name cannot start with a number or underscore (_)

  • Variable names are case-sensitive (age, Age and AGE are three different variables)

age <- 25 

Age <- 20

AGE <- 21

tot <- age + Age +AGE
tot
[1] 66
  • Reserved words cannot be used as variables (TRUE, FALSE, NULL, ifâ¦)

Here are few examples of incorrect variable naming convention:

Factor

A factor in R is used to represent categorical data. It stores data as integers with corresponding category labels called levels. Factors are useful in statistical modeling and data analysis.

colors <- factor(c("red", "blue", "red", "green", "blue"))

print(colors)
[1] red   blue  red   green blue 
Levels: blue green red
# [1] red   blue  red   green blue
# Levels: blue green red

Here:

"red", "blue", and "green" are the levels.

Internally, R stores them as integers (e.g., 3, 1, 3, 2, 1).

Use levels(colors) to see the category labels.
